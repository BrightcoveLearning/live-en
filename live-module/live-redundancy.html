Enabling Live Redundancy

Notes:

  • Every account has a SEP (Static Entry Point) limit of 3, with the exception of us-west-2 which will have a limit of 10. If you need an increase please contact Brightcove support.
  • If you are using redundancy with SSAI, you must insert ads into all jobs in the group at the same time.

The Live Redundancy feature is activated by adding two or more streams to your event in the Brightcove Live Platform.

First Create a Live event. (For more information in how to create a Live event please click here).

  1. Go to the Stream Options, these options will allow you to name and select the region of your streams. Click in the (+) Add a Backup Stream button.

    If you don't see the Add a Backup Stream button then backup streaming has not been enabled for your account.

    Note: Please contact Brightcove Support to enable this option in your account.
  2. By clicking the Add a Backup Stream button you will add a backup (redundancy) stream for your Live video. You can add up as many as your account limit supports.
    You may have redundant groups counting against your account limit. Once an event with redundant streams completes or is canceled it is no longer visible in the Live Module. However, such events can be listed and deleted via the Live API. See Live API: Redundant Groups.
  3. Name your Primary and Backup streams, and select a region that is closest to where your encoder is located. By default, the Primary stream region is set to Oregon (US West) which will produce bad results if you are streaming from another continent.

    The Reference ID is an optional identifier for internal use and must be unique (max 150 characters).

    Continue to set up your Live event as you will normally do. For more information on how to create a Live event please click here.

  4. Alternatively, you can also add a new stream from the control room window:
  5. Add your Backup Stream information and region.
  6. In the Control Room you will be able to see the Endpoint and Stream Name details of each of your streams.
  7. You can review all the streams information in the Control Room.

Note: If you created a single-stream event and you add a redundancy stream in the Control Room, you will need to republish your player embed code to benefit from redundancy.

Clipping

  1. Video Clipping will be available for each of your streams, just by selecting the desired stream from the drop-down menu. For information on creating a clip, see Creating a Video Clip Using the Live Module.
  2. Second stream selection.

Monitoring

Click in the Monitoring tab below the preview player to view monitoring information. Monitoring will be available for each of your streams, just by selecting the desired stream from the drop-down menu.

The monitoring information is available while an event is in progress and for up to 1 hour after the event ends.

Deleting a Redundant Group

With a Redundant group created:

  1. Click Cancel Stream from one job of the group.
  2. Click Stop this Stream from a second job of the group.
  3. Go to the Manage Events page and select the Completed tab.
  4. The Redundancy event should end up in Idle state.
  5. Click Close to remove the redundancy event.

Limitations

  • Live redundancy cannot be used with DRM-protected live streams.
